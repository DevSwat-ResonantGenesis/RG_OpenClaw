How Federated Polling Works

1
You click "Run" on your federated agent at dev-swat.com/agents
2
Platform queues the task in the federated_tasks table (status: pending)
3
This connector polls GET /federation/tasks/poll every 5 seconds (outbound HTTPS)
4
Task picked up → LLM-driven agent loop uses ALL platform tools (memory, search, agents, media, integrations...)
5
Result submitted via POST /federation/tasks/{id}/result → session completes in platform UI
🔒
Zero inbound connections. ALL traffic is outbound HTTPS. Works behind any firewall, NAT, or VPN. No ports exposed.
